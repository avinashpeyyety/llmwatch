llmwatch is a terminal dashboard for your Mac: memory and disk at a glance, top processes by RAM and I/O, real-time Ollama context and tok/s, and Aider API session stats for xAI and other cloud models.

What it monitors

Memory & swap

RAM and swap usage with visual bars — spot pressure before Ollama or Aider slows down.

Storage & disk I/O

Volume usage per mount point, plus per-process read/write rates for the heaviest apps.

Local LLMs (Ollama)

Loaded model RAM, context window, live generation tok/s, and last completed speed from Ollama logs.

API sessions (Aider)

Active xAI sessions: model, generating/idle status, tokens, and cost. Persists last interaction and month-to-date totals.

Install on Mac

1

Clone the repo

git clone https://github.com/avinashpeyyety/llmwatch.git
cd llmwatch
2

Run the installer

./scripts/install.sh

Creates a virtualenv in the project folder and adds llmwatch and aider-merge-stats to ~/.local/bin. Ensure that directory is on your PATH (restart Terminal or source ~/.zshrc).

3

Optional: Ollama for the local LLM panel

brew install ollama
brew services start ollama
ollama pull qwen3.5:9b   # or any model you use
4

Optional: Aider + xAI for API session stats

# Use the global aider launcher (aider-xai), not aider.real
aider-xai .

# llmwatch reads ~/.config/aider/sessions/ and api-stats.json

Usage

llmwatch                  # default 1s refresh
llmwatch --refresh 0.5    # faster updates
llmwatch --processes 20   # show more processes

Run from any directory after install. Pair with aider-xai in another terminal to populate API session stats.

  • Ollama tok/s — parsed from /opt/homebrew/var/log/ollama.log while a model generates
  • Ollama context — prompt tokens, active context, and max context from /api/ps
  • API sessions — active Aider xAI sessions with tokens, cost, and generating status
  • Last interaction — persisted in ~/.config/aider/api-stats.json after Aider exits
  • Month to date — aggregated API messages, tokens, and cost for the current month
  • Quit — Ctrl+C
