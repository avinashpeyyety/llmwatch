llmwatch is a terminal dashboard for your Mac: memory and disk at a glance, top processes by RAM and I/O, real-time Ollama context and tok/s, and OpenCode session stats for local Ollama and cloud models.

What it monitors

Memory & swap

RAM and swap usage with visual bars — spot pressure before Ollama or OpenCode slows down.

Storage & disk I/O

Volume usage per mount point, plus per-process read/write rates for the heaviest apps.

Local LLMs (Ollama)

Loaded model RAM, context window, live generation tok/s, and last completed speed from Ollama logs.

OpenCode sessions

Active OpenCode session (model, backend, status, tokens, cost) plus month-to-date totals from opencode.db.

Install on Mac

1

Clone the repo

git clone https://github.com/avinashpeyyety/llmwatch.git
cd llmwatch
2

Run the installer

./scripts/install.sh

Creates a virtualenv in the project folder and adds llmwatch to ~/.local/bin. Ensure that directory is on your PATH (restart Terminal or source ~/.zshrc).

3

Optional: Ollama for the local LLM panel

brew install ollama
brew services start ollama
ollama pull qwen3.5:9b   # or any model you use
4

Optional: OpenCode for agent session stats

brew install anomalyco/tap/opencode
opencode-xai    # or opencode-qwen for local Ollama

# llmwatch reads ~/.local/share/opencode/opencode.db

Usage

llmwatch                  # default 1s refresh
llmwatch --refresh 0.5    # faster updates
llmwatch --processes 20   # show more processes

Run from any directory after install. Pair with opencode in another terminal to populate session stats.

  • Ollama tok/s — parsed from /opt/homebrew/var/log/ollama.log while a model generates
  • Ollama context — prompt tokens, active context, and max context from /api/ps
  • OpenCode session — active session with model, tokens, cost, and generating status
  • Month to date — aggregated OpenCode messages, tokens, and cost for the current month
  • Quit — Ctrl+C
